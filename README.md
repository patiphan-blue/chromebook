# ระบบยืม-คืนและติดตาม Chromebook

โปรเจกต์นี้แยกเป็น 2 ส่วน:

- `Code.gs` คือ Backend บน Google Apps Script ใช้ Google Sheets เป็นฐานข้อมูล
- `index.html`, `app.js`, `style.css` คือ Frontend สำหรับโฮสต์บน GitHub Pages

## โครงสร้างหน้าเว็บล่าสุด

- หน้าแรกเป็น Dashboard สาธารณะ แสดงสถิติเครื่องและตารางรายการยืม
- ตารางหน้าแรกแยกเป็น `ตารางนักเรียน` และ `ตารางครู`
- แต่ละตารางแสดงรหัส, ชื่อ-สกุล, ชั้น+ห้อง/ประเภท, วันยืม, วันคืน, สถานะ, เลขเครื่อง
- ปุ่ม `เข้าสู่ระบบแอดมิน` อยู่ด้านขวาบน
- หลังแอดมินล็อกอินแล้วจะแสดงเครื่องมือ `ยืมเครื่อง`, `คืนหลายคน`, `นำเข้า Excel`

## โครงสร้างข้อมูล

เมื่อรัน `setupDatabase()` ระบบจะสร้าง 5 ชีต:

- `Config`
- `Admins`
- `Students`
- `Teachers`
- `Chromebooks`
- `Transactions`

บัญชีเริ่มต้นคือ:

- Username: `admin`
- Password: `admin123`

## แนวคิด Import Excel แบบใหม่

ตอนนี้ระบบรองรับไฟล์ Excel 2 ประเภทแยกกัน:

1. ไฟล์ข้อมูลส่วนตัวนักเรียนทั้งโรงเรียน
   - ใช้เป็นฐานข้อมูลหลักในชีต `Students`
   - ระบบอ่าน sheet แรกเป็นข้อมูลส่วนตัวหลัก
   - ถ้าแถวแรกเป็นหัวรายงาน เช่น วันเวลาที่สร้างรายงาน ระบบจะหาแถวหัวตารางจริงให้อัตโนมัติ
   - ถ้าไฟล์แยกคอลัมน์ `ชั้น` และ `ห้อง` ระบบจะรวมเป็น `ชั้น/ห้อง` เช่น `ม.1/1`
   - ถ้ามีคอลัมน์ชื่อ `เลขประจำตัวนักเรียน` ซ้ำ 2 ช่อง ระบบจะใช้ช่องแรกเป็นเลขบัตรประชาชน และช่องที่สองเป็นเลขประจำตัวนักเรียน
   - ระบบอ่าน sheet อื่น ๆ ในไฟล์เดียวกันเพื่อหาเบอร์โทร แล้วจับคู่ด้วยเลขประจำตัวนักเรียนหรือเลขบัตรประชาชน
   - ควรนำเข้าเป็นไฟล์แรก

2. ไฟล์ฐานข้อมูลเก่าการยืมปีที่แล้ว
   - ใช้สร้าง/อัปเดต `Chromebooks`
   - ใช้สร้างรายการยืมค้างใน `Transactions`
   - ไม่สร้างและไม่อัปเดตชีต `Students`
   - ถ้าเจอรหัสนักเรียนเป็น `0` หรือชื่อขึ้นต้นด้วย `ครู` ระบบจะบันทึกในชีต `Teachers` และสร้างรายการยืมเป็นประเภทครู
   - ถ้ามีรหัสนักเรียนในไฟล์ยอดยืมที่ไม่พบใน `Students` ระบบจะแสดงจำนวนเป็น `ไม่พบนักเรียน`
   - ควรนำเข้าหลังจากไฟล์นักเรียน

สาเหตุที่ก่อนหน้านี้อาจนำเข้าได้ถึงแค่บางระดับชั้น เช่น ม.5 คือโค้ดเดิมใช้ `appendRow()` ทีละบรรทัด ทำให้ Google Apps Script ทำงานช้าและเสี่ยงชน execution limit ตอนนี้ปรับเป็น `setValues()` แบบ bulk แล้ว และ Frontend ส่งข้อมูลเป็นชุดละ `400` แถว

## หัวคอลัมน์ที่รองรับ

ไฟล์ข้อมูลส่วนตัวนักเรียน:

- `เลขบัตรประชาชนนักเรียน`
- `ชั้น+ห้อง` หรือ `ระดับชั้น+ห้อง`
- `เลขประจำตัวนักเรียน`
- `คำนำหน้า+ชื่อ+สกุล`
- `วันเกิด`
- `บ้านเลขที่`
- `หมู่`
- `ตำบล`
- `อำเภอ`
- `จังหวัด`
- `เบอร์โทรศัพท์` จาก sheet อื่นในไฟล์เดียวกัน

ไฟล์ฐานข้อมูลเก่าการยืม:

- `รหัสนักเรียน`
- `ชื่อ-นามสกุล`
- `ระดับชั้น+ห้อง`
- `เลขที่`
- `เบอร์โทรศัพท์`
- `หมู่`
- `ตำบล`
- `อำเภอ`
- `จังหวัด`
- `เลขเครื่องนิยม`
- `เลขที่ทรัพย์สิน`
- `วันที่ตรวจสอบ`

Backend รองรับชื่อคอลัมน์สำรองบางแบบ เช่น `student_id`, `citizen_id`, `full_name`, `grade_level`, `Key`, `device_key`, `asset_no`

## คอลัมน์ในชีต Students

ชีต `Students` จะเก็บเฉพาะข้อมูลสำคัญ:

- `student_id`: เลขประจำตัวนักเรียน
- `citizen_id`: เลขบัตรประชาชนนักเรียน
- `grade_level`: ชั้น+ห้อง
- `student_no`: เลขที่
- `prefix`: คำนำหน้า
- `first_name`: ชื่อ
- `last_name`: สกุล
- `full_name`: คำนำหน้า+ชื่อ+สกุล
- `birth_date`: วันเกิด
- `house_no`: บ้านเลขที่
- `village_no`: หมู่
- `subdistrict`: ตำบล
- `district`: อำเภอ
- `province`: จังหวัด
- `phone`: เบอร์โทรศัพท์จาก sheet อื่น
- `address`: ที่อยู่รวมสำหรับค้น/แสดงผลเร็ว

## การยืมของครู

กรณีฐานข้อมูลยืมเก่ามีครูที่ใช้รหัสนักเรียนเป็น `0`:

- ระบบจะแยกครูไปไว้ในชีต `Teachers`
- ระบบสร้าง `teacher_id` ให้อัตโนมัติจากชื่อครู เพราะรหัส `0` ซ้ำกันหลายคน
- รายการใน `Transactions` จะมี `borrower_type` เป็น `teacher`
- หน้า `คืนหลายคน` จะมีตัวเลือก `ครู` เพื่อดูรายการครูที่กำลังยืม
- หน้า `ยืมเครื่อง` สามารถเลือกประเภทผู้ยืมเป็น `ครู` แล้วกรอกชื่อครูได้

## Deploy Google Apps Script

1. สร้าง Google Sheet ใหม่
2. ไปที่ Extensions > Apps Script
3. ลบโค้ดเดิม แล้ววางโค้ดจาก `Code.gs`
4. กด Save
5. เลือกฟังก์ชัน `setupDatabase`
6. กด Run และอนุญาตสิทธิ์
7. กด Deploy > New deployment
8. เลือกชนิดเป็น Web app
9. ตั้งค่า Execute as: Me
10. ตั้งค่า Who has access: Anyone
11. กด Deploy
12. คัดลอก Web app URL

## ตั้งค่า Frontend

เปิดไฟล์ `app.js` แล้วแก้บรรทัดแรกให้เป็น URL ของ GAS:

```js
const API_URL = "URL ของ Google Apps Script Web App";
```

ตอนนี้ในไฟล์มี URL ที่ถูกใส่ไว้แล้ว ถ้า redeploy GAS ใหม่ต้องนำ URL ใหม่มาแทน

## อัปโหลดขึ้น GitHub Pages

1. สร้าง Repository บน GitHub
2. อัปโหลดไฟล์เหล่านี้:
   - `index.html`
   - `app.js`
   - `style.css`
3. ไปที่ Settings > Pages
4. Source เลือก Deploy from a branch
5. Branch เลือก `main` และ `/root`
6. กด Save
7. รอ GitHub Pages สร้าง URL

## ลำดับการใช้งาน Import

1. Login เข้าระบบ
2. ไปที่เมนู `นำเข้า Excel`
3. เลือกไฟล์ข้อมูลส่วนตัวนักเรียนทั้งโรงเรียน
4. กด `นำเข้าฐานนักเรียน`
5. เลือกไฟล์ฐานข้อมูลเก่าการยืมปีที่แล้ว
6. กด `นำเข้ายอดยืมเก่า`
7. ไปที่ Dashboard เพื่อตรวจจำนวนเครื่องและรายการกำลังยืม

## API Actions

Frontend เรียก Backend ด้วย `POST` แบบ `text/plain` เพื่อลดปัญหา CORS preflight:

```json
{
  "action": "dashboard",
  "data": {}
}
```

Actions ที่มี:

- `login`
- `dashboard`
- `dashboardTables`
- `assignDevice`
- `listClasses`
- `getBorrowersByClass`
- `bulkReturn`
- `importStudentMaster`
- `importBorrowHistory`
- `importData` alias ของ `importBorrowHistory`
- `listAvailableDevices`
- `ping`
