# Product

## Register

product

## Users

ครูและแอดมินของโรงเรียนปายวิทยาคารที่ต้องจัดการยืม-คืน Chromebook, ตรวจสถานะเครื่อง, นำเข้าข้อมูลนักเรียน, และทำรายการจำนวนมากในช่วงเวลาจำกัด เช่น ก่อนเข้าเรียนหรือระหว่างจัดอุปกรณ์ให้ทั้งห้อง

## Product Purpose

ระบบนี้เป็นเว็บแอปสำหรับติดตาม Chromebook ของโรงเรียนโดยใช้ Google Sheets เป็นฐานข้อมูลและ Google Apps Script เป็น API ส่วนหน้าเว็บโฮสต์บน GitHub Pages เป้าหมายคือให้แอดมินเห็นสถานะเครื่องได้เร็ว ทำรายการยืม-คืนได้ถูกต้อง และลดงานซ้ำเมื่อจัดการนักเรียนทั้งห้อง

## Brand Personality

เร็ว ชัดเจน น่าเชื่อถือ ใช้โทนแดง-ขาวของโรงเรียนเป็นเอกลักษณ์ แต่หน้าตาต้องอ่านง่ายแบบเครื่องมือแอดมิน ไม่ใช่หน้าโปรโมตหรือ landing page

## Anti-references

ไม่ควรเหมือนระบบราชการเก่าที่รกและอ่านยาก, ไม่ควรมีเมนูบนแน่น ๆ จนบดบังงานหลัก, ไม่ควรใช้สีสถานะที่แยกไม่ออก, ไม่ควรตกแต่งเยอะจนแอดมินหา action สำคัญช้า, และไม่ควรเป็นหน้า hero/marketing ที่ต้องเลื่อนนานก่อนใช้งานจริง

## Design Principles

1. งานหลักต้องถึงเร็ว: แอดมินควรเข้าสู่การยืมรายห้อง ค้นหา และคืนเครื่องได้โดยไม่ต้องจำขั้นตอนมาก
2. สถานะต้องอ่านออกทันที: ว่าง, กำลังยืม, ส่งซ่อม, คืนแล้ว ต้องแยกด้วยทั้งข้อความและสีที่สม่ำเสมอ
3. รองรับข้อมูลจำนวนมาก: ตารางต้องเลื่อน ค้นหา และแบ่งกลุ่มได้ ไม่ให้ข้อมูลยาวทั้งโรงเรียนทำให้หน้าใช้งานช้า
4. แยก public view กับ admin action: หน้าแรกดูสถานะได้ทันที ส่วน action ที่เปลี่ยนข้อมูลต้องอยู่หลัง login
5. ใช้คำที่ครูเข้าใจ: ปุ่มและข้อความต้องบอกผลลัพธ์ของ action เช่น ดึงรายชื่อ, บันทึกยืมที่เลือก, คืนเครื่องที่เลือก

## Accessibility & Inclusion

ตั้งเป้าอย่างน้อย WCAG AA สำหรับข้อความและองค์ประกอบหลัก ต้องมี focus indicator สำหรับคีย์บอร์ด, ไม่สื่อความหมายด้วยสีเพียงอย่างเดียว, รองรับ reduced motion, และต้องใช้งานได้บนมือถือหรือแท็บเล็ตเมื่อครูต้องตรวจข้อมูลนอกโต๊ะทำงาน
